What is DOM?

The Document Object Model (DOM) is presentation of the document as a tree of objects.

The Document Object Model (DOM) is the presentation of the document as a tree of objects. DOM is an API for HTML and XML documents.

Example

								
									<!DOCTYPE html>
									<html>
									<head>
									  <title>MY TITLE</title>
									</head>
									<body>
									<div>
									  <h1></h1>
									  <p><a></a></p>
									</div>
									</body>
									</html>
We have a simple HTML-page w/ title in head, & div w/ children in body. * This is the DOM of our HTML-page. The root element in the hierarchy of this DOM is HTML. Inside we have HEAD and BODY, so for HEAD & BODY, HTML is a direct parent. As we can see in the picture, there are different types of nodes. Tags are represented as element nodes, and text inside tag is a text node. In general, there are 12 types of nodes, but the most commonly used are only 4 of them: element nodes, text nodes, document nodes & comment nodes.

How can we use it?

								
									document.documentElement;
								
							
								
									document.head;
								
							
								
									document.body;
Firstly we can get HTML object via document.documentElement. Also, we can obtain HEAD and BODY via document.head & document.body. DOM can be really powerful. We can add elements, we can remove elements, we can change the things inside elements. The possibilities are endless.

Elements also have own properties like id, innerHTML, childElementCount or attributes.

How to get element from DOM?

								
									document.getElementById(id);
								
							
								
									document.getElementsByTagName(tag);
								
							
								
									document.getElementsByClassName(className);
								
							
								
									document.querySelectorAll(css);
								
							
								
									document.querySelector(css);
Often, we need to get an element quickly from the DOM, wherever it is. There are different ways. One of them is getElementById(). It exists only in a document context. Other methods can be applied to any elements. querySelectorAll() & querySelector() are used to find elements by their css-selector.

Live collections

Live collections always reflect the current state of the document and “auto-update” when it changes.

Viewed methods can return live & non-live collections. All methods starting with "getElementsBy" return a live collection. Such collections always reflect the current state of the document and “auto-update” when it changes.

How to create a new element?

							
								document.createElement(tag);
To create a new element, we can use createElement() method.

Example

							
								const div = document.createElement('div');

								div.className = "smile";

								div.innerHTML = "<p>( ͡° ͜ʖ ͡° )</p>";

								document.body.appendChild(div);
Let’s see an example how it works. We pass ‘div’ value to this method to make a div element. We can change the class name or inner HTML of a new element. To add it into HTML-page, we use appendChild() method.

DocumentFragment

The DocumentFragment interface represents a minimal document object that has no parent.

The DocumentFragment interface represents a minimal document object that has no parent. It is used as a lightweight version of Document that stores a segment of a document structure comprised of nodes just like a standard document.

Other useful element properties

Elements have many useful properties. Let’s look at 2 more of them. 

elem.classList

								
									element.classList.add(string);
								
							
								
									element.classList.remove(string);
								
							
								
									element.classList.toggle(string);
classList give us the possibility to add and remove element classes. One way to do that is to call add() and remove() methods, which take class names. Another way is to call toggle() method. It also takes a class name, then it checks if this class exists. If so, it removes it, otherwise – add. So, toggle() method works like toggle-switch.

elem.style

Represents a collection of CSS property-value pairs set in elements style attribute

The last thing I want to mention is a STYLE property. It represents a collection of CSS property-value pairs set in the elements style attribute. STYLE property allows us to change values of CSS properties.
